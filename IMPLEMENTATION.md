## ✅ 1. Spaced Repetition System

### Implementation Location
`app/agents/progress.py` — `_schedule_spaced_revisions()`

### What It Does
When a user completes a topic, the system automatically schedules **future revision tasks**:

```python
Day 0: Study DBMS Normalization → complete (rating 3/5)
  ↓ automatically triggers:
Day 1: Quick recap (15 min) — auto-scheduled
Day 3: Medium revision (20 min) — auto-scheduled  
Day 7: Final checkpoint (25 min) — auto-scheduled
```

### Smart Intervals
- **Rating 1-2** (weak understanding): day+1, +3, +7 revisions
- **Rating 3-4** (moderate): day+1, +7 revisions
- **Rating 5** (mastered): day+7 only

### Key Code
```python
def _schedule_spaced_revisions(s: StudentState, task: dict, rating: int):

    if rating <= 3:

        schedule.append((7, 25, "Final checkpoint"))
    
    for day_offset, duration, label in schedule:
        available_on = (datetime.now() + timedelta(days=day_offset)).isoformat()

```

### Benefits
- **Prevents forgetting** — topics reappear automatically
- **No manual tracking** — system remembers for you
- **Adaptive** — more revisions for harder topics

## ✅ 2. Archive System (Revision-Only Mode)

### Implementation Location
`app/agents/progress.py` — `_archive_topic()`  
`app/state.py` — `knowledge_vault` field

### What It Does
**DELETE ≠ Remove Forever**

When user archives a topic:
1. ❌ Does NOT delete from system
2. ✅ Moves to `knowledge_vault`
3. ✅ Removes active study/practice tasks
4. ✅ **KEEPS all revision tasks**
5. ✅ Preserves mastery rating and notes

### Usage
```
User: archive Normalization from DBMS

🗃️ Normalization archived.
  Mastery: 3.5/5  ·  Revisions still scheduled.
  The topic lives in your Knowledge Vault — future revision tasks remain active.

User: vault

🗃️ Knowledge Vault:
  DBMS  Normalization  ★★★☆☆  (revision only)
  OS    Scheduling     ★★★★☆  (revision only)
```

### Key Code
```python
def _archive_topic(s: StudentState, topic: str, subject: str):
    # Calculate mastery from all ratings
    mastery = round(sum(ratings) / len(ratings), 1) if ratings else 0
    
    # Add to vault
    s.knowledge_vault.append({
        "subject": subject,
        "topic": topic,
        "mastery": mastery,
        "status": "archived",
        "revision_only": True
    })
    
    # Remove study/practice tasks ONLY
    s.tasks = [
        t for t in s.tasks
        if not (t.get("topic") == topic and t.get("type") in ("learning", "practice"))
    ]
    # Revision tasks remain!
```

### Benefits
- **Never lose knowledge** — vault preserves everything
- **Declutter active list** — focus on what matters now
- **Future-proof** — revisions continue automatically

## ✅ 4. Planner → Learner Bridge

### Implementation Location
`app/agents/orchestrator.py` — routing logic

### What It Does
User can **call learner agent during active task** without breaking study flow.

### Workflow
```
1. User studying: "Study DBMS Normalization" (active_task_id = 5)
   ↓
2. User types: "show me a diagram"
   ↓
3. Orchestrator detects keywords: ["diagram", "video", "mcq", "quiz"]
   ↓
4. Routes to learner agent
   ↓
5. Learner generates content (SVG + explanation + videos + MCQs)
   ↓
6. User types: "next"
   ↓
7. Returns to progress loop → continues with task
```

### Key Code
```python
def orchestrator_node(state: dict):
    # Case 2.5: Bridge — during active task, allow learner agent call
    if s.active_task_id:
        learner_keywords = ["diagram", "visualize", "video", "mcq", "quiz", "svg", "show me"]
        if any(kw in msg for kw in learner_keywords):
            s.intent = "learner"
            return s.model_dump()
```

### Trigger Keywords
- "diagram"
- "visualize"
- "video"
- "mcq" / "quiz"
- "svg"
- "show me"
- "teach me"

### Benefits
- **Seamless handoff** — no context loss
- **Just-in-time learning** — stuck? Get help instantly
- **Flexible flow** — timetable OR exploration

## ✅ 5. Adaptive Task Evolution

### Implementation Location
`app/agents/progress.py` — rating-based logic

### How It Works

#### A) Confidence-Based Promotion
```python
# After rating understanding
for subj in s.subjects:
    if rating_int >= 4:
        subj["confidence"] = min(old + 1, 10)
    elif rating_int <= 2:
        subj["confidence"] = max(old - 1, 1)
    
    # If confidence reached 7+, promote to Practice mode
    if new_conf >= 7 and old < 7:
        for t in s.tasks:
            if t.get("type") == "learning":
                t["title"] = t["title"].replace("Study ", "Practice ")
                t["type"] = "practice"
```

**Example:**
```
Day 1: Study Data Structures (confidence 5/10)
User completes 3 topics with rating 4+
Confidence → 7/10

System: "You're confident now. Remaining tasks → Practice mode"

Day 4: Practice Data Structures — Numericals & MCQs
```

#### B) Smart Practice Adaptation
```python
if rating_int >= 4:
    # High understanding → application-level practice
    practice_task["title"] = f"Practice {topic} — Numericals & MCQs"
    practice_task["recommended_resource_type"] = "practice"
    
elif rating_int == 1:
    # Low understanding → concept-first revision
    revision_task["title"] = f"Revise {topic} with Tutor"
    revision_task["type"] = "revision"
    revision_task["recommended_resource_type"] = "explanation"
```

#### C) Auto-Revision for Low Ratings
```python
if rating_int <= 2:
    # Auto-add 30-min revision task
    s.tasks.append({
        "title": f"Revise {topic}",
        "type": "revision",
        "duration_minutes": 30,
        "reason": f"Auto-scheduled: understanding rated {rating_int}/5"
    })
```

### Benefits
- **Dynamic difficulty** — adapts to user performance
- **Prevents stagnation** — promotes to practice when ready
- **Safety net** — catches struggling users early

### LLM Calls Optimized
- Planner strategy: 1 LLM call (minimal JSON)

### State Size
